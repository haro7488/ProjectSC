# AI 협업 방식

이 프로젝트에서 AI(Claude)와 어떻게 협업하는지 설명합니다.

---

## 협업 원칙

### 1. AI는 도구가 아닌 협업자
- 단순 코드 생성기로 사용하지 않음
- 설계 결정에 대한 토론 상대로 활용
- "왜?"라는 질문에 함께 답을 찾음

### 2. 모든 결정은 기록으로
- AI와의 대화에서 나온 인사이트 → DECISIONS.md
- 고민 과정 자체가 포트폴리오 가치

### 3. 컨텍스트 공유
- CLAUDE.md로 프로젝트 규칙 공유
- 스펙 문서로 현재 상태 공유
- Progress로 진행 상황 공유

---

## 협업 워크플로우

### Phase 1: 문제 정의
```
[나] 이런 기능이 필요해. 어떻게 설계하면 좋을까?
[AI] 몇 가지 접근법이 있어...
[나] A안의 장단점은?
[AI] 장점은... 단점은...
```

### Phase 2: 설계 토론
```
[나] 우리 프로젝트 컨벤션 고려하면?
[AI] CLAUDE.md 기준으로 보면 B안이 더 적합...
[나] 동의해. 근데 이 부분은 어떻게 처리하지?
[AI] 이런 방법이 있어...
```

### Phase 3: 구현
```
[나] 그럼 구현 시작하자
[AI] 먼저 스펙 문서 업데이트하고 구현할게
[AI] Progress에 작업 추가했어
```

### Phase 4: 기록
```
[AI] DECISIONS.md에 이 결정 기록할게
[나] 좋아. 나중에 왜 이렇게 했는지 알 수 있겠다
```

---

## 효과적인 AI 활용 팁

### DO
- 구체적인 컨텍스트 제공
- 여러 선택지 요청 후 비교
- "왜?" 질문으로 근거 확인
- 결정 과정 문서화 요청

### DON'T
- 코드만 요청 (설계 없이)
- 맹목적 신뢰 (검토 없이 적용)
- 컨텍스트 없는 질문

---

## 프롬프트 전략

### 설계 요청
```
"[기능]을 구현하려고 해.
현재 프로젝트 구조는 CLAUDE.md 참고해줘.
몇 가지 설계 방안을 장단점과 함께 제안해줘."
```

### 리뷰 요청
```
"이 코드/설계를 리뷰해줘.
특히 [관점]에서 개선점이 있는지 봐줘.
우리 컨벤션(CONVENTIONS.md)도 확인해줘."
```

### 문서화 요청
```
"방금 결정한 내용을 DECISIONS.md 형식으로 정리해줘.
선택지, 결정 이유, 예상 결과 포함해서."
```

## 세션 관리

### 새 세션 시작 시
1. `/clear`로 컨텍스트 정리
2. 현재 작업 상태 공유 (PROGRESS.md 참조)
3. 이번 세션 목표 명시

### 세션 종료 시
1. Progress 업데이트 확인
2. 새로운 결정 사항 DECISIONS.md 기록
3. 다음 작업 TODO 정리
