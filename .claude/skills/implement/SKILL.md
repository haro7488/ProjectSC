# 구현 프로토콜

## 실행 조건
- 스펙 문서가 존재하는 기능 구현
- 기존 패턴 기반 반복 코드
- 테스트 코드 작성

## 위임 전략

| 구현 유형 | 에이전트 | 모델 |
|-----------|----------|------|
| 기능 구현 | `general-purpose` | sonnet |
| 테스트 작성 | `general-purpose` | sonnet |
| 리팩토링 | `general-purpose` | sonnet |

## 프롬프트 템플릿

```
[기능]을 구현해줘.

참고:
- 스펙: Docs/Specs/{Assembly}.md
- 유사 패턴: {참고 파일 경로}
- 컨벤션: Docs/CONVENTIONS.md

요구사항:
- 기존 패턴과 동일한 구조
- 파일당 클래스 1개
- 네임스페이스: Sc.{폴더명}

완료 후 보고:
## 완료 보고
- 수행: (1줄)
- 결과: (생성/수정 파일 목록)
- 확인 필요: (있으면 기술)
```

- 명확한 버그 수정
